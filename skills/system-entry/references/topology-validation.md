# 拓扑校验

拓扑需要对比四类证据：

1. 管理配置。
2. OS 进程。
3. 监听端口。
4. `pgxc_node`。

## 角色映射

- GTM：全局事务管理器进程或配置项。
- CN / Coordinator：接收 SQL 连接的 PostgreSQL/OpenTenBase 协调节点。
- DN / DataNode：数据节点进程和端口。

## 一致性结果

- `consistent`：配置、进程、端口和 `pgxc_node` 基本一致。
- `partial`：至少两类证据一致，但某类证据缺失或受权限限制。
- `ambiguous`：证据互相冲突。
- `unknown`：证据不足。

## 需要报告的警告

- 配置中列出的节点没有运行。
- 进程监听端口不在配置中。
- `pgxc_node` 中的节点不可达。
- 存在多个配置文件，且无法证明哪个是当前活跃配置。
- 权限隐藏了进程所有者或端口进程名。

本技能不修复不一致问题。
