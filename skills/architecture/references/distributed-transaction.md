# 分布式事务

## DOCUMENTED

OpenTenBase 是分布式数据库。跨多个 DN 的写入或读写一致性需要事务协调，不能简单等同于单机 PostgreSQL 的本地事务。

GTM 和相关事务机制用于维护全局事务语义。具体行为需要结合 OpenTenBase 版本、配置和执行场景判断。

## 基本概念

- 单节点事务：只涉及一个 DN，协调成本较低。
- 跨节点事务：涉及多个 DN，需要全局协调。
- 只读查询：也可能需要一致的快照或全局可见性语义。
- 节点异常：可能影响事务提交、回滚或恢复。
